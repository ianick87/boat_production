Deploying Django Web Application using Cpanel.

Hi there, today we will be deploying a Django Web App using Cpanel (Shared Hosting). By End of tutorial you will be able to

  • Deploy your Django app in your shared hosting.
  • Use Git to update your application

I believe this should work on most hosting service providers that support CPanel, In my case I will be using PNRHOST.

Before we begin you must have the following

  • A ready to be deployed Django application in a git repository which includes a file requirements.txt which contains your django project dependencies.
  • Access to Terminal under Advanced in your Cpanel main page (optional).
  • Cpanel access to your Shared Hosting.

You may have to check with your service provider if setting up Python Applications is supported with your current subscription

Let’s begin!!

First let’s get our project files.

  1. Open your Cpanel and navigate to Git Version Control under the Files Section

Now let’s clone the repository with the django project, click on Create and fill in the required fields which include the git repository link and the destination path for your cloned repository, mine will be /home/enelqsub/repositories/django_project. Feel free to set it wherever you like but you must remember the path we’ll need to use it later.

2. After successfully cloning, go on to configure the Python environment, from the Cpanel main page navigate to the Software section and click on Setup Python App

Click on Create Application

First we’ll change the python version to 3.7 (depends on what you prefer)

Application Root — This is the path to your django project root folder (yes the one you cloned earlier)

Application Url — This determines how your application will be accessed, I’d recommend to leave it blank.

Application Startup File — This will determine how our app will be run. In this field type in

passenger_wsgi.py

(Bear with me, we’ll get to see what passenger_wsgi.py is about)

Application Entry Point — Setup wsgi callable object for your application. In this field type in

application

Passenger log file — This is optional but could be helpful when your app runs into some sort of error.

My final configuration looks like this

If all is set, click Create.

3. Our python environment is now set but we need to install dependencies and also prepare static files so click STOP APP if it’s running then scroll down to the part where it mentions CONFIGURATION FILES like below

In the first field just below the Run Pip Install button type in

requirements.txt

Click Add, this should activate the Run Pip Install button, click on it and select requirements.txt

Wait for it to finish and only proceed if it’s successful or else you might face fatal errors in your Django app.

Next we want to collect the static files

In the Execute Python Script type in

manage.py collectstatic —no-input

This should generate the static files required. You can also run other commands here if you need to, for example

manage.py migrate
